# Hypercert（ハイパーサート）

Hypercertは、インパクトの主張を永続的・公開的・検証可能なものにする手段です。Hypercertをミントすることで、どのような作業が行われ、誰によって行われ、どのようなインパクトが意図されていたかを証明するブロックチェーン記録を作成できます — すべてがMUSEのロジックモデルと紐付けられます。

:::info 開発中の機能
Hypercertのミントは現在、活発に開発が進められています。システムの進化に伴い、一部の機能が変更される可能性があります。本番アプリでミントを行う前に、[開発環境](https://dev.muse.beaconlabs.io) でお試しいただくことをおすすめします。
:::

## Hypercertとは何か？

Hypercertはブロックチェーン上の**インパクト証明書**です。インパクト活動の検証可能な永続的な領収書と考えてください。

- プログラムが何を達成しようとしていたかを公開記録する
- 作業に貢献した人を記録する
- 作業がいつ行われたかを指定する
- 中央の権威機関を信頼する必要なく、誰でも独立して検証できる形式ですべてを保存する

Hypercertは、インパクト認証のオープンプロトコルである[Hypercerts標準](https://hypercerts.org)上に構築されています。PDF報告書やSNS投稿とは異なり、Hypercertは作成後に変更することができません — 記録は永続的です。

### Hypercertをミントする理由

- **信頼性** — ブロックチェーン検証によってインパクトの主張が独立して検証可能になります
- **透明性** — 誰でもあなたに連絡することなくHypercertを閲覧・確認できます
- **資金調達** — 一部のインパクトファンダーや遡及的パブリックグッズの資金調達メカニズムは、作業の証明としてHypercertを認識します
- **認知** — Hypercertに記載された貢献者は、作業に対する公開・永続的な認証を受けます

## 既存のHypercertを閲覧する

すでにミントされたHypercertを見るには、[muse.beaconlabs.io/hypercerts](https://muse.beaconlabs.io/hypercerts) にアクセスします。Hypercertカードのグリッドが表示され、各カードには以下が示されています。

- Hypercertのタイトルと説明
- ロゴとバナー画像（提供されている場合）
- インパクトスコープのカテゴリ
- 貢献者のリスト

## Hypercertをミントする

ミントには接続されたウォレットが必要です。まだウォレットを接続していない場合は、先に[ウォレット接続ガイド](./wallet-connection)をご覧ください。

### ステップ1：ロジックモデルを構築する

ミントする前に、キャンバス上のロジックモデルが証明したいインパクト活動を適切に表しているか確認してください。Hypercertはインパクト主張のコンテキストとして、あなたのロジックモデルに紐付けられます。

### ステップ2：「Mint Hypercert」ダイアログを開く

1. ツールバーの **「More」** ボタンをクリックする
2. ドロップダウンから **「Mint Hypercert」** を選択する
3. Hypercertのミントフォームが開く

### ステップ3：フォームに入力する

ダイアログ左側のフィールドに入力します。入力するにつれて、右側にHypercertカードのライブプレビューが更新されます。

**タイトル（必須）**

インパクト主張の名称です。Hypercertの見出しとして表示されます。明確で具体的なタイトルにしてください。

例：
- *「農村部の若者向けコーディングブートキャンプ — コホート1（2024年）」*
- *「コミュニティガーデンネットワーク — ポートランド、オレゴン州 — 2023〜2024年」*

**説明（必須、10〜500文字）**

あなたが主張するインパクトの要約です。プログラムが何を行い、どのような変化を生み出したか、または生み出そうとしているかを説明してください。Hypercertを閲覧する誰にも表示されます。

**インパクトスコープ**

活動内容を最も適切に表すカテゴリです。一般的なカテゴリには以下があります。

- 教育
- 医療
- 環境
- 経済開発
- デジタルパブリックグッズ
- コミュニティ開発

**ロゴ（任意）**

組織のロゴをアップロードします。Hypercertカードに小さなアイコンとして表示されます。受け付けるフォーマット：PNG、JPG、SVG。推奨サイズ：200×200ピクセル以上。

**バナー画像（任意）**

Hypercertカードのビジュアルアイデンティティのためのバナー画像をアップロードします。背景またはヘッダー画像として表示されます。受け付けるフォーマット：PNG、JPG。推奨サイズ：1200×630ピクセル（横長フォーマット）。

**活動期間**

作業が行われた期間を指定します。これはHypercertが対象とする期間です — ミントする日付ではありません。

- **開始日** — プログラムや活動が始まった日
- **終了日** — プログラムや活動が終了した日（または報告期間の終わり）

**貢献者**

このインパクト活動に貢献したすべての人を、カンマ区切りで記載します。以下を含めることができます。

- 個人名（例：*「山田花子、田中一郎」*）
- 組織名（例：*「Beacon Labs、オープンソースコミュニティファンド」*）
- 匿名での記録を希望する貢献者のウォレットアドレス

ここに記載された貢献者は、Hypercertの記録に永続的・公開的な認証を受けます。

### ステップ4：プレビューを確認する

ダイアログの右側には、公開後にどのように見えるかのHypercertカードのライブプレビューが表示されます。以下を確認してください。

- タイトルと説明が明確に読めるか
- ロゴとバナー画像が正しく表示されているか（アップロードした場合）
- 貢献者のリストが完全かつ正確か

### ステップ5：ウォレットを接続する

まだウォレットを接続していない場合は、ミント前に接続する必要があります。ダイアログ内の **「Connect Wallet」** プロンプトをクリックして、[ウォレット接続ガイド](./wallet-connection) の手順に従ってください。

### ステップ6：ミントする

**「Mint Hypercert」** をクリックしてブロックチェーントランザクションを開始します。

ウォレットからトランザクションの確認と関連するガス代の承認を求められます。**開発環境**（dev.muse.beaconlabs.io）では、実際のコストなしに無料のテストETHが使用されます。**本番アプリ**では、実際のガス代が発生します。

ウォレットで確認後、ミントプロセスが開始されます。トランザクションがオンチェーンで確認されるまでお待ちください。完了すると、HypercertがHypercertsギャラリーに表示されます。

## ガス代

| 環境 | ネットワーク | 費用 |
|------|------------|------|
| [dev.muse.beaconlabs.io](https://dev.muse.beaconlabs.io) | Base Sepolia（テスト用ネットワーク） | 無料（フォーセットからテストETHを取得） |
| [muse.beaconlabs.io](https://muse.beaconlabs.io) | 本番環境 | 実際のガス代が発生 |

:::tip まずは開発環境で練習する
Hypercertのミントが初めての場合は、開発環境から始めてください。SepoliaフォーセットからテストETHを無料で取得し、コストゼロでミントプロセス全体を体験して、本番環境でミントする前にすべてが正しく見えるか確認できます。テストETHの取得方法については[ウォレット接続ガイド](./wallet-connection)をご覧ください。
:::
